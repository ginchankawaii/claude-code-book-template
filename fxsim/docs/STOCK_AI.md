# AI日本株・信用買い（Opus＋kabuステーションAPI）

FX版と同じOpus頭脳で、**日本株の信用買い（買いのみ）**を全自動化。Opusが
**候補銘柄の決算・ニュース・マクロをweb検索**して買い銘柄を選定し、確信度で
レバレッジ（株数）を決める。最大3銘柄。執行は **auカブコム kabuステーションAPI**。

> ⚠️ FXのMT5以上に証券会社アプリ依存。**必ず最初は「検証環境」**で動かすこと。

## オフライン体験（`--sim`・口座/kabuステーション不要）
auの審査待ちで本物のkabuステーションがまだ無くても、**仕組みだけ**今すぐ画面で見られる。
合成の株価（実ティッカー）＋メモリ内の信用ポートフォリオで、選定→株数算出→発注→
建玉→評価損益→ダッシュボードまで一通り回る。**実データ・実約定ではないシミュレーション**。
```powershell
cd C:\Users\penan\claude-code-book-template\fxsim
git pull
# 8サイクル回す（APIキー無し＝ルールベース選定でOK）
py -3.12 -m scripts.run_stock_ai --sim --sim-steps 8 --exclude 7518,9719,8053
# 別ウィンドウでダッシュボード
py -3.12 -m app.server          # http://localhost:8000/live
```
- `ANTHROPIC_API_KEY` を入れておくと、**シミュ株価の上でも“本物のOpus選定”**（決算/ニュースを
  web検索して実在銘柄を選ぶ）が動く。キー無しなら無料のルールベース選定で仕組みだけ確認。
- あくまで動作確認用。**実運用の検証は審査通過後にkabu検証環境(18081)で**。

## 仕組み
```
kabuステーション(常駐) ──REST──▶ Python(Opus判断・サイジング) ──REST──▶ kabuステーション(発注)
  ・ランキング/板/余力/建玉を取得      ・候補をOpusがweb検索→買い銘柄選定        ・信用新規買い/返済売り
```
- 候補プール：kabuのランキング（売買代金・値上がり率）＋`--watch`指定銘柄＋保有銘柄
- サイズ：1銘柄 **最大5%リスク**・**単元(100株)単位**・**信用余力の範囲内**・1枠~総資産の34%上限
- 安全弁：dry-run既定／最大3銘柄／ドローダウン時に自動減量／エラー時は全保有維持

## セットアップ（Windows）
1. **auカブコム証券の口座**＋**信用取引口座**を開設。
2. **kabuステーション**をインストールして起動・ログイン。
   - メニューから **API設定 → APIを利用する**（**まず「検証」**を選択）。**APIパスワード**を設定。
   - 検証環境はポート **18081**、本番は **18080**。
3. `fxsim\.env` に追記：
   ```
   ANTHROPIC_API_KEY=sk-ant-...
   KABU_API_PASSWORD=（APIパスワード）
   KABU_ORDER_PASSWORD=（取引/注文パスワード）
   ```
4. 依存は既存のまま（`requests` 利用、追加インストール不要）。

## 使い方
```powershell
cd C:\Users\penan\claude-code-book-template\fxsim
git pull

# 検証環境・ドライラン（Opusに聞くが発注しない）
py -3.12 -m scripts.run_stock_ai --once --dry

# 検証環境・実発注（検証口座なので安全）
py -3.12 -m scripts.run_stock_ai --once --live

# 常駐（1日1回・検証環境・ドライラン）
py -3.12 -m scripts.run_stock_ai

# 本番に切替（十分検証してから！）
py -3.12 -m scripts.run_stock_ai --prod --live
```
- `--watch 7203,6758` で候補に銘柄を追加できる。
- `--exclude 7518,9719,8053` で **インサイダー規制等の売買禁止銘柄を完全除外**（候補から外し、
  万一AIが選んでも発注前に弾く二重防御）。勤務先・グループ銘柄は必ず指定すること。
- ダッシュボード：`py -3.12 -m app.server` → `http://localhost:8000/live`
  （判断ログに **銘柄コード・BUY/HOLD/CLOSE・株数・確信度・理由・根拠・保有方針** が出る）

## パラメータ
| 引数 | 既定 | 意味 |
|---|---|---|
| `--max-risk` | 0.05 | 1銘柄の最大リスク |
| `--max-positions` | 3 | 同時保有の最大銘柄数 |
| `--candidates` | 20 | Opusに渡す候補数（コスト/質のバランス） |
| `--watch` | "" | 候補に足す銘柄（カンマ区切り） |
| `--live` / `--prod` | off | 実発注 / 本番ポート（既定は検証・dry-run） |

## 正直な注意
- **バックテスト不可**（AI＋決算＋市場スキャン）。検証＝**検証環境で数週間フォワード**。
- 信用取引特有のリスク：**追証・金利・貸株料・6ヶ月期限・逆日歩**（売りは今回なし）。max-risk/銘柄数で抑制。
- kabuのAPIは時間帯・銘柄で発注が抑止されることがある（コード100368等）。失敗ログが出たら一緒に対処。
- まず `--dry` でOpusの選定を見て、`--live`（検証）で約定確認 → 納得後に `--prod`。

## ダッシュボードはFX/信用株で分離済み
`http://localhost:8000/live` の上部に **「FX」「信用株」の切替ボタン**がある。
それぞれ最新のFX run / 株 run を独立表示するので、両方同時に動かしても混ざらない。
URL直指定なら `…/live?kind=stocks`（株）/ `?kind=fx`（FX）。特定runは `?run_id=` も可。
（MT5ブリッジの建玉ステータスはFX表示のときだけ反映され、株表示には影響しない）
