# 夜間リサーチ報告：「勝てる仕組み」を探した結果

依頼：「何度試してもいいから勝てる仕組みを作って」。一晩、systematicに探索した
正直な記録。派手な結論を盛らず、効いたものと効かなかったものを両方残す。

## 方法（過剰最適化を避けるための作法）
- データ：ドル円 **日足55年（1971–2026, 14,190本）** ＋ **15分足1ヶ月（2026年5月, 1,988本）**。
- **ウォークフォワード**：パラメータは訓練期間で選び、**別の検証期間（アウトオブサンプル）**でのみ採点。
- **コストを必ず計上**（スプレッド、約定コスト）。複数のスプレッド水準で頑健性も確認。
- 指標は `scripts/research*.py` に再現可能な形で保存。

## 効かなかったもの（重要）
| 手法 | 結果 | なぜ |
|---|---|---|
| 平均回帰（RSI/ボリンジャー） | OOSで**マイナス**（全期間） | 逆張りは現代相場で負ける。あなたが溶かしたタイプ |
| M15の機械的売買 | **1ヶ月で −37〜39%** | 月100回超の過剰売買でスプレッドに殺される |
| Claude判断（hybrid） | 取引0＝**±0%** | 弱いシグナルを全部見送り。損は防ぐが“儲け”は生まない |
| 分散モメンタム・アンサンブル | 現代では買い持ちに**勝てず**、5pipsでエッジ消滅 | 単独では薄い・コスト敏感 |

→ 「AIに任せれば勝てる」「機械的指標で勝てる」はいずれも**幻想**だと定量確認。

## 効いたもの：日足・長期トレンドフィルター（ロング＆待機）
**「終値が長期移動平均（SMA100〜200）より上の時だけロング、下なら待機」**。
ショートはしない（円安の構造バイアスに逆らうと負ける）。

本物のエンジン（コスト・ATRストップ・サイジング込み）での検証：

| 期間 | 機械的rule | 買い持ち | **TREND(SMA150)** |
|---|---|---|---|
| 2010+ | −37% / Sharpe −1.0 | +133% / 1.87 | **+255% / Sharpe 2.68** |
| 2015+ | −42% / −1.5 | +27% / 0.89 | **+62% / 1.45** |
| 2020+ | −18% / −0.7 | +48% / 1.63 | **+108% / 2.35** |

特徴：
- **買い持ちを上回るリスク調整リターン**、かつ**市場にいる時間は約55%**（暴落を回避）。
- **低頻度**（年 数回〜十数回）＝コスト耐性が高い（スプレッド5pipsでもほぼ不変）。
- **SMA100〜250で一貫してプラス**＝特定値への過剰最適化ではない（SMA50だけ速すぎて弱い）。

## 盛りすぎ防止：2つのモデルの食い違い（必読）
上の「+255% / Sharpe 2.68」は**本物のエンジン（ATRリスク・サイジング＝トレンドに
リスクを乗せる＋勝ちを伸ばす）**の数字で、サイジングが利益を増幅している。
より保守的な**単純vol-targetモデル**＋**SMA自体もOOSで選ぶ**最も厳しい検証では：

| 2010+ | CAGR | Sharpe | 最大DD |
|---|---|---|---|
| 買い持ち | +4.1% | +0.44 | 31.5% |
| トレンド（SMAもOOS選択） | +3.5% | +0.47 | 30.1% |
| トレンド・アンサンブル（無選択・SMA100-250平均） | +3.6% | **+0.50** | **24.8%** |

→ 正直な実像：**リターンは買い持きと同等、Sharpeは僅かに上、最大の効果は
「ドローダウンを2〜3割減らす＝暴落回避」**。ATRトレーリングを足しても改善せず。
**“倍々で増える必勝法”ではない。リスク効率と低頻度・低コストが本質。**
実際の成績はポジションサイズの取り方に大きく依存し、攻めるほどDDも比例して増える。

## 正直な注意（ここを読んで）
- **勝率は低い（18〜38%）**：トレンド追随は「小さく何度も負け、たまに大きく勝つ」。**精神的にキツい**。
- **ドローダウンは20〜34%**ある。無痛ではない。
- **単一通貨・単一サンプル**。2012–2015と2021–2024の**歴史的円安トレンドに乗った**結果でもある。同じ相場が続く保証はない。Sharpe 2台はこの強トレンド期間ゆえの上振れ。
- これは**日足スイングのエッジ**。15分スキャルでは（コストで）再現しない。
- **必勝法ではない**。「機械売買より遥かにマシで、買い持ちよりリスク効率が良い、低頻度の規律的システム」。

## 攻めの設定：レバレッジと時間足を最適化した結果（research_run7 / _run8）
「もっと攻めたい・5倍レバまで許容・日足にこだわらず短い足でもいい・複利で最大化」
という依頼への**正直な実測**。`max_leverage`（`FXSIM_MAX_LEVERAGE`）で
**ノーション/エクイティの上限を固定**する。リスク%がサイズを押し上げるノブ、
レバ上限がハード天井（兼・安全装置）。全て本物のエンジン（コスト＋ATRストップ＋
レバ上限）で計測＝本番サイジングと一致。

### ① レバを上げるほど複利が伸びる、は誤り（最重要）
USD_JPY 日足トレンド（SMA150・TPなし・5x上限）、**2000年以降26年**：

| リスク% | ピークレバ | CAGR | 最大DD | 26年の倍率 |
|---|---|---|---|---|
| 2% | 3.6x | 4.2% | 41% | 2.9x |
| **3%** | **5.0x** | **5.4%** | **55%** | **4.0x** |
| **4%（新既定）** | **5.1x** | **5.6%** | **65%** | **4.1x** |
| 5% | 5.1x | 5.2% | 74% | 3.8x |
| 6% | 5.1x | 4.1% | 80% | 2.8x |
| 8% | 5.2x | 2.3% | 86% | 1.8x |

→ **CAGRはリスク3〜4%（ピークレバ約5倍）で頭打ち**、それ以上はDDだけ膨らみ
**CAGRはむしろ低下**する（＝ボラティリティドラッグ／過剰レバの複利破壊。大きなDDが
複利の土台を削る）。「5倍が最大、それ以上は無理」という直感は**数学的に正しい**。
最適点は `risk 4% / 5x 上限`。これを新既定にした。

### ② 期間で全く違う（レジーム依存）
同じ `SMA150 / risk4% / 5x`：

| 期間 | CAGR | 最大DD | 倍率 |
|---|---|---|---|
| 2000+ | 5.6% | 65% | 4.1x |
| 2010+ | 12.2% | 45% | 6.5x |
| 2015+ | 5.9% | 44% | 1.9x |
| 2020+ | 19.7% | 40% | 3.1x |

直近窓の12〜20%は**2012-15・2021-24の歴史的円安に乗っただけのレジーム運**。

### ③ ウォークフォワード（盛り防止の本番）
訓練2000-2014で最良パラメータ（SMA250/risk4%, 訓練CAGR 7.8%）を選び、
**OOS 2015-2026で採点 → CAGR 3.0% / DD 53% / 倍率1.4x**。
→ **インサンプル最適はOOSで再現しない**。正直な期待値は**CAGR 3〜8%／DD 40〜65%**。
SMAは100〜250どこでもプラスなので、特定値に最適化しない（するとこの罠にはまる）。

### ④ 短い足は本当に負けるのか → 11年の実データで再検証（research_run8）
当初はM15が1ヶ月分しか無く「短い足はコストで負ける」と結論したが、その後
**MT5から M15/M30/H1/H4 の11年分（2015-01〜2026-06）を取得**。結論が変わった。
`risk4% / 5x / TPなし`、全期間2015-2026の各足ベスト：

| 足 | SMA | ≒ルックバック | CAGR | 最大DD | 取引/11年 |
|---|---|---|---|---|---|
| 日足 | 90 | 90日 | 9.5% | 38% | 60 |
| H4 | 400 | 67日 | 11.2% | 39% | 187 |
| **H1** | **2400** | **100日** | **12.4%** | **34%** | **219** |

→ **約2〜3ヶ月のトレンドフィルターをH1で回すのが最良**で、日足(9.5%)を上回る。
エッジの正体は*ルックバックの長さ*が主、ただし**同じ66日でも 日足SMA66=7.1% vs
H4 SMA400=11.2%** で、細かい足の方が良い（トレンド内の入り/出のタイミング＋
イントラバー・ストップが効く）。

**コスト頑健性（スプレッド0.8→2.5pips）**：H1 SMA2400は `12.4%→11.7%→10.8%`。
取引219回（≈年20回）と低頻度なので**スプレッドを上げてもエッジは消えない**（＝
過剰売買のコスト負けとは別物）。1ヶ月時点で見たM15の−9%は、足が細かすぎて
取引が100超に膨らんだのが原因で、**“短い足”そのものではなく“過剰売買”が悪**だった。

**ウォークフォワード（訓練2015-2020 → OOS 2021-2026）**：

| 足 | 訓練CAGR | OOS CAGR | OOS DD |
|---|---|---|---|
| 日足 SMA90 | −4.7% | 23.9% | 27% |
| H4 SMA500 | 0.4% | 29.0% | 47% |
| **H1 SMA2400** | −2.1% | **26.7%** | 31% |

→ イントラデイはOOSでも日足以上。**ただし訓練2015-2020は全構成がマイナス〜横ばい**。
利益の正体は2022-2024の円暴落で、**ロング限定トレンドはレンジ期に必ず痩せる**。
レジーム無敵のエッジではない。

### 結論（攻めの最適解）
`H1トレンド / SMA2400（≒100日）/ risk4-5% / 5x上限 / TPなし`。
全期間 **CAGR 12.4% / DD 34%**、OOSで26.7%、スプレッド2.5pipsでも10.8%。
リスク掃引でもCAGRはrisk5%で頭打ち（5倍キャップが拘束＝それ以上は無意味で安全）。
**資金は無から増えない**：¥10万口座なら年¥1万強。「月に数万」は口座資金そのものを
増やす話で、レバ5倍は既に複利の天井。45%級のDDではなく**34%のDDに耐える前提**。

## 使い方（攻め：H1 トレンド）
```bash
FXSIM_STRATEGY=trend          # トレンドフィルター（唯一の堅牢なエッジ）
FXSIM_GRANULARITY=H1          # H1（11年検証で日足を上回る・コスト頑健）
FXSIM_TREND_SMA=2400          # ≒100日トレンド。1800〜2400が堅い（特定値に最適化しない）
FXSIM_RISK=0.04               # 攻め。5%でも可だが頭打ち（5xキャップが効く）
FXSIM_MAX_LEVERAGE=5          # ハード上限＝攻めのダイヤル兼安全装置
FXSIM_USE_TAKE_PROFIT=0       # 早すぎる利確を切り、勝ちを伸ばす（トレンド用）
```
**保守的に日足**で回すなら `FXSIM_GRANULARITY=D / FXSIM_TREND_SMA=90`（CAGR 9.5%/DD38%）。
任意で、ライブ時はイベント回避（±60分ブラックアウト）を重ねると指標スパイク事故を防げる。
※ H1/H4の生データは `fxsim/data/USD_JPY_H1.csv`・`_H4.csv` に同梱（M15/M30は容量の都合で除外）。

## 結論
あなたの失敗（機械的M15で溶かす）の真因は **過剰売買とコスト**。
夜通し探した答えは劇的な必勝法ではなく、**「低頻度・長期トレンド・暴落回避」という地味だが頑健な規律**。
実弾の前に、必ず**フォワードのペーパートレードで数ヶ月**確認してください。
```
再現：python3 -m scripts.research_run / _run2 / _run3 / _run4 / _run5
攻めのレバ（日足・55年）：python3 -m scripts.research_run7
攻めの時間足（11年・H1勝者）：python3 -m scripts.research_run8
```
