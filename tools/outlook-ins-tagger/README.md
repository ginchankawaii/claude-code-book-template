# Outlook INS番号アシスタント — 工数登録の毎日15分を数分に

Outlookの予定にINS番号(クラウドログの案件番号)を自動付与し、
INS番号ごとの工数集計を表示するPowerShellスクリプトです。

**Pythonのインストール不要。** Windows標準のPowerShellとOutlookだけで動くので、
そのまま誰にでも配布できます。

## 狙い

クラウドログには「OutlookにINS番号を入れておけば一発連携できる」公式機能が
既にありますが、事前にINS番号を入れる手間がネックで使われていません。
このスクリプトはその**入口の手間だけ**を自動化します。社内システム
(クラウドログ/HCM)を直接操作しないので、壊れにくく、規程面でも安全です。

さらに、INS番号ごとの合計時間を表示するので、連携機能を使わない人でも
「昨日の工数の答え」を見ながらクラウドログに入力できます。

## セットアップ(初回のみ)

1. このフォルダを自分のPCにコピー
2. `ins_dictionary.sample.csv` をコピーして `ins_dictionary.csv` を作成
3. 自分の案件に合わせてキーワードとINS番号を書く(上の行ほど優先)

   | Keyword | Ins | Note |
   |---|---|---|
   | ○○社定例 | 12345 | ○○社NW更改案件の定例会議 |
   | △△検証 | 23456 | △△案件の検証作業 |
   | 社内MTG | 90001 | プレ工数(社内会議) |

   ※Excelで編集する場合は「CSV UTF-8(コンマ区切り)」形式で保存してください。

## 使い方

```powershell
# 昨日の予定をドライラン(表示のみ・何も書き込まない)
powershell -ExecutionPolicy Bypass -File .\Set-InsNumber.ps1

# 実際にOutlookへ書き込む
powershell -ExecutionPolicy Bypass -File .\Set-InsNumber.ps1 -Apply

# 先週分をまとめて処理
powershell -ExecutionPolicy Bypass -File .\Set-InsNumber.ps1 -Date 2026-07-21 -Days 5
```

出力例(ドライラン):

```
=== 2026/07/23 から 1 日分 [ドライラン(表示のみ)] ===
開始         時間  INS    状態       件名
----         ----  ---    ----       ----
07/23 09:00  0.5   90001  付与(予定)  社内MTG 朝会
07/23 10:00  2     12345  付与(予定)  ○○社定例
07/23 14:00  3     -      未マッチ    ファームウェア調査

--- INS番号ごとの工数集計 (クラウドログ入力用) ---
INS    合計時間  件数
---    --------  ----
12345  2         1
90001  0.5       1
```

「未マッチ」が出たら辞書にキーワードを足していきます。
数日で自分の業務パターンをほぼカバーできるはずです。

## パラメータ

| パラメータ | 既定値 | 説明 |
|---|---|---|
| `-Date` | 昨日 | 対象開始日 |
| `-Days` | 1 | 対象日数 |
| `-Target` | `Category` | `Category`=分類に付与 / `SubjectPrefix`=件名の先頭に付与 |
| `-InsFormat` | `INS:{0}` | 付与する文字列の書式 |
| `-Apply` | なし | 付けたときだけ実際に書き込む(既定はドライラン) |

## ⚠️ 導入前に確認すること

- **クラウドログのOutlook連携が、INS番号を予定の「どの欄・どの書式」から
  読むのか**を確認し、`-Target` と `-InsFormat` を仕様に合わせてください。
  仕様が分かるまでは、書き込みせず工数集計のチートシートとして使うだけでも
  毎日の入力時間を短縮できます。
- 辞書には案件名・顧客名が入るため、個人PC内で管理し社外に出さないこと。

## 制限

- 終日予定はスキップします(工数対象外の想定)
- 定期的な予定(繰り返し)は各回に展開されて処理されます
- 予定が重複している時間帯の按分はしません(集計は単純合計)
